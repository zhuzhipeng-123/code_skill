# Skill Manager — 技能管理器

## 执行流程

每次触发时，按以下步骤执行：

1. 运行 `ls ~/.claude/skills/` 获取当前已安装的技能目录名列表
2. 与下方缓存的目录名列表对比：
   - **一致** → 直接输出缓存的技能列表，无需读取任何文件
   - **不一致** → 执行"刷新缓存"流程，更新本文件后再输出

**缓存的目录名列表**（用于校验）：
```
context-switch-v1.0.0
skill-creator-v1.0.0
skill-evolution-manager-v1.0.0
skill-manager-v1.0.0
url-to-skill-v1.0.0
```

## 缓存的技能列表（校验通过后直接输出）

---

你目前共安装了 **5 个技能**：

1. **模型切换器** (`context-switch-v1.0.0`)
   描述：无缝切换 AI 模型并保留对话上下文。支持智能摘要、一键切换、上下文恢复功能。触发词：`/switch`、`切换模型`、`换模型`、`switch model`。

2. **技能创建器** (`skill-creator-v1.0.0`)
   描述：用于创建新技能、修改和改进已有技能，以及评估技能性能。支持从零构建技能、更新优化已有技能、运行评测和性能基准测试。

3. **技能进化管理器** (`skill-evolution-manager-v1.0.0`)
   描述：专门用于在对话结束时，根据用户反馈和对话内容总结优化并迭代现有 Skills 的核心工具。它通过吸取对话中的"精华"（如成功的解决方案、失败的教训、特定的代码规范）来持续演进 Skills 库。

4. **技能管理器** (`skill-manager-v1.0.0`)
   描述：管理和查看所有已安装的 Claude Code 技能，直接从缓存输出列表，无需扫描文件。

5. **URL 转技能** (`url-to-skill-v1.0.0`)
   描述：将任意 GitHub 仓库链接或文档网站 URL 转换为结构化的 Claude Code 技能文件，自动抓取、分析并打包内容保存到本地。

---

## 刷新缓存（校验发现目录不一致时，或用户明确要求"刷新"/"重新扫描"时执行）

1. 用 `ls ~/.claude/skills/` 列出所有子目录
2. 对每个目录读取 `SKILL.md` 的 frontmatter，提取 `name` 和 `description`
3. 将结果翻译为中文，**更新本文件中"正常展示"部分的缓存列表**
4. 输出更新后的列表给用户

## 注意事项

- 有新技能安装后（如通过 `url-to-skill` 或 `skill-creator`），应提示用户运行"刷新技能列表"以更新缓存。
- 如果 `~/.claude/skills/` 目录为空或不存在，告知用户目前没有安装任何技能。
