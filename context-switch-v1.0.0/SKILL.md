# Context Switch - 模型切换器（自动上下文恢复版）

## 触发条件

当用户表达**切换模型的意图**时触发此技能，包括但不限于：

### 固定命令
- `/switch`
- `切换模型`
- `换模型`
- `switch model`

### 带模型名称的表达（自动识别模型）
- `切换到[模型名]` / `切换到 [模型名]`
- `换成[模型名]` / `换成 [模型名]`
- `用[模型名]` / `用 [模型名]`
- `给我切换到[模型名]`
- `把模型改成[模型名]`
- `帮我切换到[模型名]`
- `我想用[模型名]`
- `改成[模型名]`

### 可识别的模型名称别名
| 用户输入 | 对应模型 |
|----------|----------|
| glm / glm-5 / 智谱 | glm-5 |
| qwen / 千问 / 通义 | qwen3.5-plus |
| kimi / 月之暗面 | kimi-k2.5 |
| minimax / MiniMax | MiniMax-M2.5 |

**注意**：如果用户指定的模型名称不在列表中，则展示模型选择列表让用户选择。

---

## 功能说明

此技能实现**无缝模型切换**：
1. 切换前：保存完整上下文到文件
2. 重启后：新会话自动加载上下文
3. 加载后：自动清理上下文文件

**用户体验**：输入 `/switch` → 选择模型 → 重启 → 自动恢复，无需任何手动操作。

---

## 执行步骤

### 步骤 1：选择目标模型

如果用户已指定模型（如"切换到glm"），直接识别；否则使用 AskUserQuestion 让用户选择：

| 模型 ID | 显示名称 |
|---------|----------|
| glm-5 | 智谱 GLM-5 |
| qwen3.5-plus | 通义千问 3.5 Plus |
| kimi-k2.5 | 月之暗面 Kimi K2.5 |
| MiniMax-M2.5 | MiniMax M2.5 |

### 步骤 2：保存完整上下文

将完整对话上下文保存到 `~/.claude/context-switch/pending-restore.md`，格式如下：

```markdown
# 上下文恢复文件

> ⚠️ 此文件由 context-switch 技能自动生成，加载后会自动删除

**切换时间**: [YYYY-MM-DD HH:MM]
**原模型**: [旧模型ID]
**目标模型**: [新模型ID]
**工作目录**: [当前工作目录路径]

---

## 用户偏好

[从 MEMORY.md 读取的用户偏好信息]

---

## 完整对话历史

### 用户消息记录

[按时间顺序列出所有用户消息，保留完整内容]

### 助手关键回复摘要

[提取关键回复的核心内容，避免过长]

---

## 关键上下文

### 主要任务
[用户正在做什么]

### 关键决策
[做出的重要决定]

### 代码变更
[创建或修改的文件列表]

### 未完成事项
[待处理的任务]

---
*此文件将在新会话加载后自动删除*
```

### 步骤 3：更新配置

修改 `~/.claude/settings.json` 中的 `ANTHROPIC_MODEL` 字段为目标模型。

### 步骤 4：提示重启

输出提示信息：

```
✅ 模型切换准备完成！

**目标模型**: [新模型名称]
**上下文已保存**: 完整对话历史已保存

请执行以下操作：
1. 输入 `/exit` 退出当前会话
2. 重新启动 Claude Code
3. 新会话将自动恢复上下文，无需任何操作
```

---

## 自动恢复机制（新会话启动时）

### 检测触发

每次会话开始时，检查 `~/.claude/context-switch/pending-restore.md` 是否存在：
- **存在** → 执行自动恢复流程
- **不存在** → 正常启动

### 自动恢复流程

1. **读取上下文文件**
   使用 Read 工具读取 `~/.claude/context-switch/pending-restore.md`

2. **解析并恢复上下文**
   - 恢复用户偏好
   - 加载对话历史
   - 确认关键任务和未完成事项

3. **删除上下文文件**
   使用 Bash 工具删除文件：
   ```bash
   rm ~/.claude/context-switch/pending-restore.md
   ```

4. **输出恢复确认**
   ```
   ✅ 上下文已自动恢复！

   **原模型**: [旧模型]
   **切换时间**: [时间]

   我们继续之前的工作吧！
   ```

## 文件结构

```
~/.claude/
├── context-switch/
│   └── pending-restore.md    # 待恢复的上下文（加载后自动删除）
├── skills/
│   └── context-switch-v1.0.0/
│       └── SKILL.md          # 本技能文件
└── settings.json             # 包含模型配置
```

---

## 注意事项

1. **上下文文件生命周期**：创建 → 重启后加载 → 自动删除
2. **多次切换**：如果用户连续切换多次，新的上下文会覆盖旧的
3. **上下文丢失**：如果用户重启后不恢复（直接开始新对话），文件会保留，下次启动仍可恢复
4. **手动清理**：用户可以手动删除 `pending-restore.md` 放弃恢复
